6 Orchestration Primitives.
When to use each?
💬 Direct / Goal 📋 Skills 🤖 Subagents ⚡ Dynamic Workflows 🔁 Loop 👥 Agent Teams
Most people use one.
Claude Code gives you six — each with a fundamentally different
shape, cost profile, and right scenario.
Temporal vs Spatial Parallel vs Sequential Repeatable vs One-off
💬
Direct / Goal
Single context. No agents spawned.
Claude handles everything inline.
Use when
Task fits in one context. A skilled person could do it in 5 minutes. Simple, focused, one-shot.
● Lowest cost Instant
Examples
prompt
Explain this regex: /^(?=.*[A-Z]).{8,}$/
prompt
Draft a client update email about the delay
prompt
Summarize this PR description in 3 bullets
Not for
Tasks that overflow a context window or need to run again next week
📋
Skills
Loads a behavioral template.
Claude follows it. No agents spawned.
Use when
You have a repeatable process — same checklist, same structure, same rigor every invocation.
● Low cost ✓ Repeatable
Built-in skills
/code-review5-dimension structured review
/security-reviewOWASP checklist
/specRequirements + acceptance criteria
/commitConventional commit format
Create your own: /skill-creator → ~/.claude/skills/
🤖
Subagents
One isolated agent handles one task.
Your context stays clean.
Use when
The task would pollute your context with logs, searches, file reads — delegate and get a crisp summary back.
●● Medium Context-isolated
How to trigger
@-mention (guaranteed)
@code-reviewer look at the auth changes
Another agent
@data-scientist analyze this CSV and summarize
Custom agent file
.claude/agents/my-agent.md
Not for
10+ parallel workers — use a Workflow instead
⚡
Dynamic Workflows
JS script orchestrates N agents in parallel.
Resumable. Session stays responsive.
Use when
Large-scale parallel work — 200 file audits, 170-claim verification, 500 component migrations, 10-company research.
●●●● High (3–10× direct) Max 1,000 agents
How to trigger
"workflow" keyword
Run a workflow to audit every endpoint
in src/routes/ for missing auth checks.
Built-in research workflow
/deep-research [topic]
Auto-workflow mode (whole session)
/effort ultracode
Session
✓ Stays responsive
Resumable
✓ Same session
🔁
Loop
Same task on a schedule.
Temporal recurrence — not parallelism.
Use when
Time drives the repetition — monitoring, polling, recurring reports, daily habits. The task itself is not parallel.
● × N iterations ✓ Self-pacing
Trigger patterns
Fixed interval
/loop 2m /check-build-status
Daily recurring
/loop 24h /morning-brief
Self-paced (model decides interval)
/loop /babysit-prs
👥
Agent Teams
Lead agent coordinates
specialist workers. Role-based.
Use when
Complex project where specialists need persistent roles over many turns — not just a one-shot data pipeline.
●●● High Experimental
Teams vs Workflows
Teams
Persistent specialist roles
Lead coordinates peers
Role-based collaboration
Long-running projects
Workflows
Data pipeline pattern
Script-driven fan-out
Parallel + synthesize
Resumable, repeatable

The Core Insight
The shape of work determines the primitive.
Loop
⏱
Temporal — one thing,
many times over a clock
≠
Workflow
⚡
Spatial — many things,
at the same moment
≠
Skill
📋
Repeatable — same process,
same rigor, every time
Direct = simple + focused
Subagent = isolated + clean
Teams = specialist + persistent
Decision Framework
5 questions. One answer.
Would hiring 20 interns help with this task?
→
⚡ Dynamic Workflow
Will you run this exact process again next week?
→
📋 Skill
Does it repeat on a clock, not in parallel?
→
🔁 Loop
Would this flood your main context window?
→
🤖 Subagent
None of the above — just do this one thing now.
→
💬 Direct

Anti-patterns
The 3 most common mis-matches.
Workflow for a simple question
✗ Wrong
Run a workflow to draft a tweet
✓ Right
Draft a tweet about our new feature
If 1 person could do it in 5 min → Direct
Direct for 200-file analysis
✗ Wrong
Read all files in src/ and find every bug
✓ Right
Run a workflow to audit all files in src/
If it won't fit in one context → Workflow
Loop when you need parallelism
✗ Wrong
/loop 0s process next item in queue
✓ Right
Run a workflow to process all 50 in parallel
Loop = temporal. Workflow = spatial.
