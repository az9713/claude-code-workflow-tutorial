  ---
  Agents in Claude Code

  Agents are isolated Claude sessions with their own system prompts, tools, and   permissions. They work independently and return results to the main
  conversation.

  ---
  The @agent-name syntax

  Guarantees a specific agent handles a task — like @-mentioning a file:

  @code-reviewer review the auth module
  @data-scientist analyze sales data

  A typeahead picker appears, you select the agent, and it's guaranteed to run.
  Plugin agents use scoped names like @agent-my-plugin:code-reviewer.

  ---
  Two ways to spawn agents

  ┌────────────────────┬────────────────────────────────────────────────────┐
  │      Approach      │                    How it works                    │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ Agent tool         │ Claude decides when to delegate — when a side task │
  │ (automatic)        │  would flood context                               │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ @-mention          │ You guarantee a specific agent runs                │
  │ (explicit)         │                                                    │
  └────────────────────┴────────────────────────────────────────────────────┘

  ---
  The claude-code-guide agent specifically

  It's a built-in lightweight agent (Haiku model) that Claude automatically
  delegates to when you ask "how do I use Claude Code?" questions. You don't
  invoke it directly — Claude routes to it based on question type.

  ---
  Creating custom agents

  Run /agents → Library → Create new agent. Or write files manually to:
  - ~/.claude/agents/ — personal, available everywhere
  - .claude/agents/ — project-level only

  Full docs: https://code.claude.com/docs/en/sub-agents.md

  ---
  Claude Code Dynamic Workflows — Complete Tutorial

  1. Core Concept

  A dynamic workflow is a JavaScript orchestration script that Claude writes to
  run tens to hundreds of parallel subagents in a single coordinated task. The
  key architectural difference:

  ┌──────────────┬──────────────────────────────────────────────────────────┐
  │   Approach   │                    Who holds the plan                    │
  ├──────────────┼──────────────────────────────────────────────────────────┤
  │ Regular chat │ You guide turn-by-turn                                   │
  ├──────────────┼──────────────────────────────────────────────────────────┤
  │ Subagents    │ Claude decides next steps each turn                      │
  ├──────────────┼──────────────────────────────────────────────────────────┤
  │ Workflows    │ A JS script holds all loops, branching, and coordination │
  └──────────────┴──────────────────────────────────────────────────────────┘

  This unlocks scalability (hundreds of agents) and repeatability (save and
  rerun the script).

  ---
  2. How to Trigger a Workflow

  Three ways:

  1. The workflow keyword — just include it in your prompt:
  Run a workflow to audit every API endpoint for missing auth checks
  2. /deep-research — built-in workflow for investigation tasks
  3. /effort ultracode — Claude automatically decides when to use workflows
  throughout the session (costs significantly more)

  Before it runs, Claude shows you the script for approval. Press Ctrl+G to open
  it in your editor first.

  ---
  3. Workflow vs. Regular Claude Code

  ┌──────────────────┬────────────┬──────────────────┬─────────────────────┐
  │      Aspect      │  Regular   │    Subagents     │      Workflows      │
  ├──────────────────┼────────────┼──────────────────┼─────────────────────┤
  │ Session          │ Blocked    │ Blocked          │ Stays responsive    │
  │ responsiveness   │            │                  │                     │
  ├──────────────────┼────────────┼──────────────────┼─────────────────────┤
  │ Scalability      │ One turn   │ A few tasks      │ Dozens to hundreds  │
  ├──────────────────┼────────────┼──────────────────┼─────────────────────┤
  │ Repeatability    │ Ask again  │ Nondeterministic │ Same script, same   │
  │                  │ each time  │                  │ orchestration       │
  ├──────────────────┼────────────┼──────────────────┼─────────────────────┤
  │                  │            │                  │ Yes — cached        │
  │ Resumable        │ No         │ No               │ results survive a   │
  │                  │            │                  │ pause               │
  └──────────────────┴────────────┴──────────────────┴─────────────────────┘

  ---
  4. Budget & Token Control

  Workflows are 3–10x more expensive than turn-by-turn conversation. How to
  control cost:

  - Test on a small slice first before running the full scope
  - Monitor with /workflows — shows per-agent token usage in real time
  - Stop without losing work — completed agents are cached; resume picks up
  where you left off
  - Runtime caps: 16 concurrent agents, 1,000 agents max per run

  ---
  5. Six Domain Examples

  ---
  Software Engineering — Security Audit

  Task: Scan every API endpoint for missing authentication checks across a large
  codebase.

  Fan-out:
  - Phase 1: One agent per endpoint file checks for missing requireAuth(), bad
  role validation, token gaps
  - Phase 2: Independent verifier agents attempt to exploit findings to confirm
  severity
  - Phase 3: Reporter synthesizes a risk-ranked list

  Output: CRITICAL: POST /api/admin/users missing auth — any user can create
  admin accounts

  Why a workflow: 200 files sequentially = 30+ minutes. In parallel = 3–5
  minutes, with a reusable script for every future commit.

  ---
  Research & Fact-Checking — Investigative Report

  Task: Research whether AI code generation has reduced software engineer hiring
  demand.

  Fan-out:
  - 5 agents search different angles simultaneously (hiring trends, AI adoption,
  salary data, job postings, company headcount)
  - Cross-check agents compare claims across sources
  - Consensus vote on what survived verification
  - Reporter synthesizes only verified claims

  Output: Balanced cited report distinguishing confirmed findings from hype,
  with sources linked

  Why a workflow: Single-agent research is one pass with no adversarial check.
  Parallel + cross-check = trustworthy output.

  ---
  Personal Productivity — Quarterly Review

  Task: Generate a Q2 personal review from calendar, email archive, and project
  log.

  Fan-out:
  - Agent 1: Calendar → key meetings, travel, conferences
  - Agent 2: Email → high-volume contacts, recurring projects
  - Agent 3: Project notes → shipped items, blockers
  - Analyst agent: Synthesizes themes (growth, relationships, skills)
  - Challenge agent: "Did you actually ship that? Was it really a win?"
  - Writer agent: Polished quarterly review with confidence score

  Output: Balanced review distinguishing real wins from rationalized ones —
  scored and grounded in data

  ---
  Business — Competitive Intelligence

  Task: Analyze 10 competitors in the AI coding tool space.

  Fan-out:
  - 10 agents in parallel, one per competitor (pricing, features, user
  sentiment, go-to-market)
  - Sentiment agents fetch Reddit/Product Hunt/review sites
  - Strategist synthesizes a competitive matrix and identifies white space

  Output: Feature comparison table + "White Space Opportunities" section
  identifying gaps none of the competitors fill

  ---
  Creative & Media — Story Outlining from Multiple Angles

  Task: Outline a sci-fi novel about AI and humanity.

  Fan-out:
  - 5 agents brainstorm from different perspectives simultaneously: AI's POV,
  protagonist's POV, antagonist's logic, supporting cast subplots, thematic
  backbone
  - Conflict agent identifies tensions between perspectives
  - Unifier synthesizes a coherent 3-act structure honoring all angles

  Output: Full story outline with thematic depth that a single-perspective
  brainstorm would miss

  ---
  Finance — Portfolio Risk Analysis

  Task: Analyze a portfolio across 6 risk dimensions.

  Fan-out:
  - 6 agents in parallel: currency, concentration, liquidity, interest rate,
  counterparty, geopolitical risk
  - Correlation agent identifies how risks interact
  - Stress test: "What happens if 3 risks hit simultaneously?"
  - Strategist recommends hedges

  Output: Risk scorecard + stress test results + Most Dangerous Interaction
  section (e.g., concentration × geopolitical risk)

  ---
  6. When to Use vs. When NOT to Use

  Use workflows when:

  - Work is parallelizable at scale ("would hiring 20 interns help?")
  - Quality requires cross-checking (research, security audits)
  - The process is repeatable (you'll run it again — save the script)
  - You need background execution while doing other work
  - You have budget and tested on a small slice first

  Don't use workflows when:

  - The task is simple — just ask Claude directly
  - You need user input between stages — workflows don't pause for approvals
  mid-run
  - The task is inherently sequential — parallelization doesn't help
  - Cost is critical — one agent can handle it
  - You'd want to retarget mid-run — stop, edit the script, relaunch instead

  ---
  Quick Start

  1. Type: "Run a workflow to [your task]"
  2. Review the script (Ctrl+G to open in editor)
  3. Approve
  4. Monitor with /workflows
  5. Save: in /workflows → select run → press s → save to .claude/workflows/
  6. Reuse: /your-workflow-name via autocomplete

  Official docs: https://code.claude.com/docs/en/workflows
